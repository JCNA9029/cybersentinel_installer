# CyberSentinel — Complete Feature Testing Guide

# Run every test from inside the CyberSentinel project folder.
# cd C:\CyberSentinel

================================================================================
 BEFORE YOU START — SETUP CHECKLIST
================================================================================

1. You are in the correct folder:
      cd C:\CyberSentinel

2. Delete the old database so cache hits don't interfere with tests:
      del threat_cache.db
   (You can recreate it by running any scan — it auto-creates on startup.)

3. Ollama is running in the background.

4. Your API keys and webhook are configured (Option 11 in the menu).

5. Python dependencies are installed:
      pip install -r requirements.txt

 FEATURE 10 — ATTACK CHAIN CORRELATION (Menu Option 6)
================================================================================

PURPOSE: Test that the correlator detects multi-step attack sequences.
We seed the event_timeline table manually to simulate a chain.

STEPS:
  1. Open a NEW Command Prompt (separate from your main terminal)
  2. Navigate to the CyberSentinel folder
  3. Run this Python snippet to inject test events:

     python -c "
     import sqlite3, datetime
     db = 'threat_cache.db'
     now = datetime.datetime.now().strftime('%Y-%m-%d %H:%M:%S')
     with sqlite3.connect(db) as c:
         c.execute('INSERT INTO event_timeline (event_type,detail,pid,timestamp) VALUES (?,?,?,?)',
                   ('LOLBIN_ABUSE', 'certutil.exe download', 0, now))
         c.execute('INSERT INTO event_timeline (event_type,detail,pid,timestamp) VALUES (?,?,?,?)',
                   ('C2_CONNECTION', '185.220.101.5 — python.exe', 0, now))
         print('Test events injected.')
     "

  4. Go back to your main terminal running CyberSentinel.py
  5. Select: 6

EXPECTED OUTPUT:
  ═══════════════════════════════════════════════════════════════
    🔴  ATTACK CHAIN: Process Injection → C2
    Severity : CRITICAL
    MITRE    : T1055 — Process Injection
    Details  : LoLBin abuse followed by outbound C2...
  ═══════════════════════════════════════════════════════════════

PASS CRITERIA:
  ✓ Chain alert fires with correct chain name
  ✓ Severity shown as CRITICAL
  ✓ MITRE mapping displayed

TEST BENIGN (no chain):
  Delete threat_cache.db or wait until the 10-minute window expires.
  Select Option 6 again.
  Expected: "[*] No attack chains detected yet."

 FEATURE 12 — FILELESS / AMSI ALERTS (Menu Option 8)
================================================================================

PURPOSE: View the AMSI alert history table.

NOTE: This tab will show "No fileless alerts detected yet" UNLESS:
  1. pywin32 is installed (pip install pywin32)
  2. PowerShell Script Block Logging is enabled (see SOC Dashboard Manual)
  3. The daemon has been running and observed PowerShell execution

TO TEST IT FULLY (requires Administrator + pywin32):
  Step 1 — Enable ScriptBlock logging (run PowerShell as Administrator):
    Set-ItemProperty "HKLM:\Software\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging" `
      -Name "EnableScriptBlockLogging" -Value 1 -Force

  Step 2 — Start the daemon:
    python CyberSentinel.py --daemon "C:\CyberSentinel\test_samples"

  Step 3 — In a separate PowerShell window, run a test obfuscated command
    (this is harmless but matches the detection pattern):
    powershell -Command "[System.Convert]::FromBase64String('dGVzdA==')"

  Step 4 — Back in CyberSentinel, select: 8

  EXPECTED:
    AMSI ScriptBlock alert showing "[System.Convert]::FromBase64String" pattern

MINIMUM TEST (no pywin32):
  Select: 8
  EXPECTED: "[*] No fileless alerts detected yet."  (correct — means it ran fine)
  PASS CRITERIA: No crash.

================================================================================
 IF SOMETHING FAILS — FIRST CHECKS
================================================================================

Webhook not firing:
  → Check Option 11 — is webhook URL set?
  → Is it HTTPS? Discord/Slack URLs always start with https://
  → Try scanning fresh (delete threat_cache.db) — cache hits need fix from latest zip

Quarantine prompt not appearing:
  → Make sure you are using the latest zip (CyberSentinel_v1_latest.zip)
  → Delete threat_cache.db and rescan
